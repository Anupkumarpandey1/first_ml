# 🏥 HealthAI Diagnostics

> An AI-powered full-stack health report analysis platform — upload any lab report, get a structured medical breakdown, and chat with an AI doctor that knows your full health context.

---

## ✨ What It Does

1. Patient registers with basic info (name, age, gender, height, weight)
2. Uploads a blood test / CBC / lab report (PDF or image)
3. Gemini Vision reads every parameter from the report
4. Gemini generates a structured, plain-language analysis
5. An AI doctor chat (powered by a LangGraph ReAct agent + RAG) answers follow-up questions with full awareness of the patient's report

## 🏗️ Architecture & Logic

### High-Level Data Flow

```
User fills patient form
    └─► POST /patient/register
        └─► Session created in memory (UUID key)
            BMI computed, patient_info stored

User uploads lab report
    └─► POST /report/upload  (multipart/form-data)
        ├─► PDF?  →  pypdf extracts text
        └─► Image? →  base64 encoded
            └─► Gemini 2.5 Flash Vision
                └─► Returns JSON: { "Hemoglobin": { value, unit, reference_range, status }, ... }
                    └─► Stored in session["report_text"]

    └─► POST /report/analyze
        └─► Gemini 2.5 Flash (text)
            Prompt = patient profile + lab JSON
            └─► Returns structured JSON:
                { summary, good_findings, concerning_findings,
                  disease_risks, follow_up_tests, lifestyle_tips }
                └─► Stored in session["report_analysis"]

User chats with Dr. HealthAI
    └─► POST /chat
        └─► LangGraph ReAct Agent
            ├─► Tool: patient_profile_retrieval
            │       reads session → patient info + report + analysis
            └─► Tool: medical_literature_search
                    FAISS similarity search → top 2 guideline chunks
            └─► Gemini generates Markdown response
                └─► Rendered in UI via react-markdown
```

### Backend

#### FastAPI (`main.py`)

The HTTP layer. Handles CORS, static file serving, and routes all requests. Sessions live in a plain Python `dict` keyed by `uuid4()` — no database needed.

| Endpoint | Method | What happens |
|----------|--------|-------------|
| `/patient/register` | POST | Validates patient info, computes BMI, creates session, returns `session_id` |
| `/report/upload` | POST | Reads file bytes, calls Gemini Vision, validates it's a health report, stores extracted JSON |
| `/report/analyze` | POST | Sends patient profile + extracted JSON to Gemini, stores structured analysis |
| `/chat` | POST | Runs the LangGraph agent with the user message and session history |
| `/patient/predict` | POST | Legacy endpoint — runs sklearn Logistic Regression for diabetes risk |
| `/static/correlation.png` | GET | Serves the ML correlation heatmap |

---

#### Gemini Vision — Report Extraction (`agent.py → extract_report_with_vision`)

**Library:** `google-genai`  **Model:** `gemini-2.5-flash`

- PDF files → `pypdf.PdfReader` extracts all page text → sent as a text prompt part
- Images → base64-encoded → sent as `inline_data` with the correct MIME type
- Gemini is prompted to return `NOT_A_HEALTH_REPORT` for non-medical files (validated server-side → HTTP 400)
- Valid reports → JSON object where every key is a test name with `value`, `unit`, `reference_range`, `status`

---

#### Gemini Analysis — Report Interpretation (`agent.py → analyze_report_with_gemini`)

**Library:** `google-genai`  **Model:** `gemini-2.5-flash`

Constructs a prompt combining the full patient profile and extracted lab JSON. Gemini returns a structured JSON:

```json
{
  "summary": "Plain-language 2-3 sentence overview",
  "good_findings": [{ "parameter", "value", "explanation" }],
  "concerning_findings": [{ "parameter", "value", "severity", "explanation", "what_to_do" }],
  "disease_risks": [{ "disease", "risk_level", "explanation" }],
  "follow_up_tests": [{ "test_name", "reason", "urgency" }],
  "lifestyle_tips": ["tip 1", "tip 2"]
}
```

Severity values: `mild / moderate / serious`  
Risk levels: `low / moderate / high`  
Urgency values: `routine / soon / urgent`

---

#### RAG Pipeline (`rag_pipeline.py`)

**Libraries:** `langchain-community`, `langchain-text-splitters`, `faiss-cpu`, `sentence-transformers`

**Building the index (one-time):**
1. Medical guidelines text is written to `sample_medical_guidelines.txt` (diabetes, hypertension, hyperlipidemia treatments)
2. `TextLoader` → LangChain `Document` objects
3. `RecursiveCharacterTextSplitter` → 300-char chunks, 50-char overlap
4. `HuggingFaceEmbeddings` (`all-MiniLM-L6-v2`) → dense vectors, runs fully locally
5. `FAISS.from_documents` → vector store saved to `faiss_index/`

**Querying:**
- Index loaded from disk with the same embedding model
- Returns top 2 most semantically similar chunks per query (`k=2`)

---

#### LangGraph ReAct Agent (`agent.py → run_agent`)

**Libraries:** `langgraph`, `langchain-google-genai`, `langchain-core`

A ReAct (Reason + Act) agent created with `langgraph.prebuilt.create_react_agent`. Initialized once and cached.

**LLM:** `ChatGoogleGenerativeAI` — `gemini-2.5-flash`, temperature 0.3

**Tools the agent can call:**

| Tool | What it does |
|------|-------------|
| `patient_profile_retrieval` | Reads the current session — returns patient info, raw report text (first 3000 chars), and structured analysis |
| `medical_literature_search` | Runs a FAISS similarity search against medical guidelines, returns top 2 matching chunks |

**System prompt** tells the agent to: always retrieve the patient profile first, use Markdown formatting, explain things simply, be compassionate, and always recommend a real doctor for serious concerns.

**Per-request flow:**
1. Last 10 messages from `chat_history` are prepended as context
2. Agent reasons → calls tools as needed → generates final response
3. Response extracted from the last message object
4. Both user message and AI response saved back to `chat_history`

---

#### ML Pipeline (`ml_pipeline.py`)

**Libraries:** `pandas`, `numpy`, `scikit-learn`, `seaborn`, `matplotlib`, `joblib`

One-time training script — not imported at runtime.

1. Generates 1000 synthetic patient records: `Age`, `Blood_Pressure`, `Cholesterol`, `BMI`, `Glucose`
2. Injects 5% random NaN values → fills with column medians
3. Computes a weighted `risk_score` from threshold rules → binary `Diabetes_Risk` label (1 if score > 3.0)
4. Plots a `seaborn` correlation heatmap → saved as `correlation.png`
5. 80/20 train-test split → `StandardScaler` → `LogisticRegression`
6. Serializes model + scaler with `joblib` → `risk_model.pkl`, `scaler.pkl`

### Frontend

#### React 19 + Vite 8

All UI lives in a single `App.jsx`. State is lifted to the root `App` component and passed down via props/callbacks.

```
App  (step, sessionId, patientInfo, extractedData, analysis)
├── StepBar           — 4-step progress bar with connecting lines
├── PatientIntake     — step 1: registration form + live BMI calculator
├── UploadReport      — step 2: drag-and-drop upload → calls /report/upload + /report/analyze
└── AnalysisDashboard — steps 3+4: tabbed analysis panel + AI doctor chat
```

**StepBar** — renders steps 1–4 with active/completed visual states driven by the `current` prop.

**PatientIntake** — controlled form, computes BMI live as `weight / (height/100)²`, color-codes the category (Underweight / Normal / Overweight / Obese), POSTs to `/patient/register`.

**UploadReport** — drag-and-drop zone (also click-to-browse), accepts `.pdf .png .jpg .jpeg .webp`. On click:
1. POSTs file as `multipart/form-data` to `/report/upload`
2. On success, POSTs to `/report/analyze`
3. Advances to step 3 with extracted data + analysis

**AnalysisDashboard** — split two-panel layout:

- Left panel — 5 tabs:
  - Overview: stat cards (counts of findings) + lifestyle tips + correlation heatmap
  - Good Findings: normal parameters with plain explanations
  - Concerns: abnormal parameters with severity badges + action advice
  - Risks: disease risks with risk-level badges
  - Follow-up Tests: recommended tests with urgency badges

- Right panel — AI Doctor Chat:
  - Messages rendered with `react-markdown` for full Markdown support
  - Quick-question buttons shown on first load
  - Auto-scrolls to latest message via `useRef` + `useEffect`
  - POSTs to `/chat` with `session_id` + `message`

## 🚀 Running Locally

### Backend

```bash
cd backend
pip install -r requirements.txt

# One-time setup — only needed if .pkl / faiss_index don't exist
python ml_pipeline.py
python rag_pipeline.py

# Set your Gemini API key (get one free at https://aistudio.google.com/app/apikey)
export GOOGLE_API_KEY=your_key_here   # Linux/Mac
set GOOGLE_API_KEY=your_key_here      # Windows

uvicorn main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at `http://localhost:5173`, backend at `http://localhost:8000`.

## ☁️ Free Deployment

### Backend → [Render](https://render.com) (free tier)

1. Connect this repo on Render → New Web Service
2. Set root directory to `backend` (or use the included `render.yaml` for auto-config)
3. Add environment variable: `GOOGLE_API_KEY=<your key>`
4. Render runs: `uvicorn main:app --host 0.0.0.0 --port $PORT`

### Frontend → [Vercel](https://vercel.com) or [Netlify](https://netlify.com) (free tier)

1. Import this repo → set root directory to `frontend`
2. Add environment variable: `VITE_API_URL=https://your-backend.onrender.com`
3. Build command: `npm run build` — output directory: `dist`

> The API URL is read from `import.meta.env.VITE_API_URL` at build time, falling back to `http://localhost:8000` for local dev.
